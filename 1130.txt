@파일 입출력

-자바 입출력 스트링 (InputStream/outputStream)
자바 프로그래밍이 디스크에 접근해서 파일을 읽고 쓰는
통로를 스트림이라고 한다

1. 스트림은 입출력 장치와 자바프로그램에 양끈ㅌ에서 전달된다
2. 단방향 통신이며 입력/출력 스트림은 독립적으로 각각움직인다
3. 연속된 데이터의 흐름으로 입출력 진행시 다른 작업을 할수 없게
상태가된다(입출력이 인터럽트의 최우선 단위가 된다 )

@입출력 스트림의 종류
입출력 패키지는 java.io 클래스 에서 제공되며 파일시스템의 정보를 \
얻기 위해서 File 클래스와
데이터 입출력을 위한 다양한 스트림
클래스를 제공한다

1. 바이트 기반의 스트림
파일형식에 구애 받지않고모든 파일을 읽고 쓸수 있으며
바이트 기반 입력 스트림의 최상의 클래스는 inputStream 이며
바이트 기반 출력 스트림의 최상위 클래스는 OutputStream 이다

1) InputStream
상속을 받아서 다야양한 스트림을 제공하고 있으며 개발환경과
상황에 따라 목적을 갖고 다양한 메소드를 오버라이딩 해서 
계층 구조를 사용하게된다
#read() : 문자를 1byte 씩 읽고 반환이 없으면 -1을 반환
#read(byte[] b) :  배열에 읽을 문자를 저장하고 읽은 수 만큼 반환 없으면 -1
#read(byte[], int offset, int len)
: 주어진 배열에 정해진 범위만큼을 알아서 저장한다
offset : 시작위치 len : 길이
#int available() : 스트림으로 읽어올수 있는 데이터 크기를 반환
#close() :  사용을 종료 자원을 반환


fileInputStream
파일에서 바이트 단위로 자료를 읽어 들일때 사용하는 스트림이다
이미지, 동영상, 텍스트 모든 타입의 파일들을 읽어올수있다
1)파일을 반복하는 횟수를 줄일수 때는 버퍼의 길이만 큼 ㅍ전체의 
값을 줄려줄수 있다
read():1byte->read(byte[] b) : 배열의 값을 늘려서 버퍼의 크기를 크게한다 
큰바구니로 한꺼번에 담아서 쓰겠다


FilterInputStream: DataInputStream/BufferedInputStream
ByteArrayInputStream
ObjectInputStream


2) OutputStream
바이트 기반의 최상위 클래스 OutputStream 객체로 제공되며
상속 다양한 출력 스트림이 존재한다
 FileOutputStream
파일을 버퍼단위로 읽어 들이면서 여러가지 파일들의
낸용을 출력하는 방법을 사용한다
new FileOutputStream(  경로/파일명/이어쓰기 옵션)
이어쓰기 옵션 : true / false : 새로만든다
이미지를 복사해서 패키지 안에서 새롭게 이미지를 생성하는 방법
read(byte[]) :  단위로 묶어야 큰 파일은 성능이 빨라진다

2. 문자 기반 스트림
문자를 기반으로 하기 때문에 텍스트 문서만으로 쓸수있다 
문자기반 입력 스트림의 최상의 클래스  Reader 가 있으며 문자 기반
출력 스트림의 최상위 클래스는  writer이다

1)Reader : 문장 입력 스트림
BufferedFeader
CharArrayReader
InputStreamReader >  FileReader

`
2) Writer : 문자 출력 스트림 
문자기반의 출력스트림 이고 최상위 클래스는  Writer 상속해
다양한 클래스를 제공한다
BufferedWfite
CharArrayWrite
InputStreamWriter :  FileWriter


3. 보조 스트림
보조스트림은 실제로 데이터를 주고받지않고 스트림의 기능을 
향상 시키거나 새로운기능을 제공해주는 스트림으로다른 보조 스트림과 
중첩하여서 사용한다
-기반 스트림 : 대상에 직접적으로 자료를 읽고 쓰는 스트림이다
-보조 스트림 :" 직접 읽고 쓰는 기능은 없지만 기반 스트림에 추가로 사용하는
스트림이다

1) 성능 향상 보조 스트림
하드디스크나 네트워크 입출력은 성능에 영향을 주기 때문에
보조스트림을 이요해 버퍼에 메모리 작업을 시켜서 성능을 향상 시킬수 있다
-바이트 기반
BufferdInputStream/BufferOutPutStream(바이트기반)
프로그램 장치로부터 데이터를 직접 읽는 대신 버퍼에 저장해 두었다가
큰단위로 받을 숭 있고 출력에 대한 보조스트림은 프로그램에서 전송한 \
데이터 를 내부 버퍼에 쌓아 두었다가 꽈차면 모든 데이터를 한꺼번에 저장한다 

- 문자기반
BufferdReader/BufferdWriter(문자기반 )
